# 2.5. Hàm theo dõi huỷ lệnh

**Tên hàm:** `order_book.track_order_book_changes(ticker_config=ticker_config, action="add", accumulate_window=60, callback=process_data)`

**Mục đích:**
Theo dõi các thay đổi bất thường trong sổ lệnh theo từng bước giá, bao gồm hành vi thêm lệnh hoặc huỷ lệnh. Hàm hỗ trợ phát hiện các tín hiệu bất thường theo thời gian thực để phục vụ giám sát giao dịch và phân tích hành vi thị trường.

**Cách hoạt động:**
1. Theo dõi bám theo giá cố định:
   - Mỗi mã được theo dõi ở mức giá cụ thể: `bid2.price` hoặc `ask2.price` (tuỳ theo `side`) ngay tại thời điểm khởi đầu.
   - Trong quá trình theo dõi, hệ thống chỉ quan sát khối lượng tại đúng mức giá đó.

2. Ngưng theo dõi khi giá vượt giới hạn số bước giá hiển thị:
   - Nếu mức giá ban đầu bị đẩy ra khỏi vùng nhìn thấy:
     - Với HOSE: ngoài 3 bước giá tốt nhất (ví dụ: từ `bid1` đến `bid3`)
     - Với HNX/UPCoM: ngoài 10 bước giá tốt nhất
   - Hệ thống sẽ ngưng theo dõi ticker đó để tránh nhiễu dữ liệu.

3. Xử lý giá `bid1` / `ask1`:
   - Giá `bid1` / `ask1` sẽ được trừ khối lượng đã khớp để tránh nhầm lẫn giữa huỷ lệnh và khớp lệnh thực tế.

4. Tính chỉ số OTR:

`OTR = Số lượng lệnh (đặt, sửa, hủy) / Số lượng giao dịch thực hiện`

**Tham số (Parameters):**

| Tên tham số | Kiểu dữ liệu | Mô tả |
|---|---|---|
| ticker_config | dict | Từ điển chứa từng mã chứng khoán và ngưỡng khối lượng thay đổi (`min_volume`) cần cảnh báo |
| side | str | `'bid'`, `'ask'` hoặc `'all'` – chỉ định hướng theo dõi |
| action | str | `'cancel'`, `'add'` hoặc `'all'` – chỉ định loại thay đổi lệnh muốn theo dõi |
| accumulate_window | int | Đơn vị giây. Cho phép user định nghĩa tần suất tính lại để cộng tổng các giá trị huỷ/thêm lệnh |
| min_action_volume | int | Khối lượng cổ phiếu huỷ/thêm tối thiểu để được tính là 1 lệnh huỷ/thêm. Phục vụ thống kê số lệnh huỷ/thêm |
| callback | function | Hàm do người dùng tự định nghĩa để xử lý dữ liệu cảnh báo khi có tín hiệu bất thường |

**Kết quả trả về (Returns):**
- Kiểu dữ liệu: dữ liệu cảnh báo theo từng sự kiện dưới dạng `dict`
- Mỗi bản ghi bao gồm các trường: `symbol`, `signal_type`, `bid_price` hoặc `ask_price`, `add_volume`, `add_number`, `match_number`, `time_start`, `time_end`
- Dữ liệu được truyền vào hàm `callback` để người dùng tự xử lý
- Hỗ trợ phát hiện các tín hiệu như thêm lệnh bất thường, huỷ lệnh bất thường và phục vụ phân tích OTR

**Ví dụ sử dụng:**

```python
from FiinQuantX import FiinSession

username = "REPLACE_WITH_YOUR_USERNAME"
password = "REPLACE_WITH_YOUR_PASSWORD"

client = FiinSession(username=username, password=password).login()

order_book = client.OrderBook()

ticker_config = {
    "VJC": {"min_volume": 100_000},
    "VNM": {"min_volume": 100_000},
    "VPB": {"min_volume": 100_000},
    "VRE": {"min_volume": 100_000}
}

def process_data(data: dict):
    print(data)

# Theo dõi hủy lệnh bất thường
order_book.track_order_book_changes(
    ticker_config=ticker_config,
    action="add",
    accumulate_window=60,
    callback=process_data
)
