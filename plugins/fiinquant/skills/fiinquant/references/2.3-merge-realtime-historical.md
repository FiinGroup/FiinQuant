# 2.3. Hàm nối dữ liệu Realtime và lịch sử

Mô tả cách sử dụng thư viện sau khi người dùng đã đăng nhập. Các ví dụ chi tiết được nêu ra ở cuối chương này.

**Tên hàm:** `client.Fetch_Trading_Data(...)`

**Mục đích:**
Kết hợp dữ liệu lịch sử và realtime theo cùng một luồng xử lý. Ở lần gọi đầu tiên, thư viện lấy dữ liệu lịch sử phù hợp với timeframe người dùng yêu cầu; khi `realtime = True`, thư viện tiếp tục kết nối websocket để cập nhật dữ liệu theo thời gian thực và tổng hợp các timeframe tiếp theo từ dữ liệu realtime.

**Cơ chế:**
- Tại thời điểm người dùng gọi lần đầu, thư viện sẽ lấy dữ liệu lịch sử phù hợp với timeframe người dùng gọi.
- Khi thông số `realtime = True`, thư viện sẽ kết nối dữ liệu qua websocket, và cập nhật data theo thời gian thực (realtime), các timeframe tiếp theo sẽ được tổng hợp từ dữ liệu realtime.

**Cảnh báo và lưu ý:**
-Do các timeframe được cập nhật theo thời gian thực, việc duy trì kết nối ổn định sẽ đảm bảo dữ liệu được tổng hợp chính xác nhất. Đối với kết nối realtime, khuyến khích người dùng sử dụng đường truyền qua mạng có dây (LAN) và đường internet ổn định.

-Trường hợp cần đồng bộ với data từ server, user có thể xây dựng cơ chế gọi lại hàm khi cần để lấy lại dữ liệu lịch sử tổng hợp trên server.

- Do có một độ trễ nhỏ giữa server và phía thư viện nên khi thực thi code nối dữ liệu realtime, người dùng sẽ cần phải chờ một chút để dữ liệu được đồng bộ và chính xác.
- Khi truyền `period` ở chế độ `realtime = True`, `period` là số nến lịch sử. Ví dụ ở lần gọi đầu tiên với `period = 100` thì sẽ là 100 nến gần nhất, với nến thứ 100 là nến realtime vẫn đang được cập nhật. Khi sang các nến khác thì dữ liệu trả về sẽ nhiều hơn 100.

**Tham số (Parameters):**  
*(Lưu ý: `period` chỉ tồn tại khi không truyền `from_date` và ngược lại.)*

| Tên tham số | Mô tả | Kiểu dữ liệu | Mặc định | Bắt buộc |
|---|---|---|---|---|
| realtime | Có đăng ký vào dữ liệu update liên tục hay không hay chỉ gọi dữ liệu lịch sử đến thời điểm mới nhất (True là có, False là không). | bool |  | Có |
| tickers | Danh sách mã được viết in hoa. | str hoặc list |  | Có |
| fields | Các trường dữ liệu cần lấy `['open','high','low','close','volume','bu','sd']` ứng với Open, High, Low, Close, Volume, BU, SD. | list |  | Có |
| adjusted | Giá điều chỉnh hay chưa điều chỉnh (True là đã điều chỉnh và ngược lại). | bool | True | Không |
| callback | Là hàm do người dùng tự định nghĩa để thao tác với dữ liệu. | function |  | Có |
| by | Đơn vị thời gian (1m, 5m, 15m, 30m, 1h, 2h, 4h, 1d). | str | 1M | Không |
| period | Số nến lịch sử gần nhất cần lấy. | int |  | Không |
| from_date | Mốc thời gian lấy dữ liệu xa nhất. | str | datetime | Không |
| to_date | Mốc thời gian lấy dữ liệu gần nhất. | str | datetime.now() | Không |
| wait_for_full_timeFrame | Chờ hết nến mới gọi callback hay không. True là chờ hết cây nến, False là cập nhật liên tục. | bool | False | Không |

**Kết quả trả về (Returns):**
- Kiểu dữ liệu: object kết nối dữ liệu lịch sử và realtime
- Class xử lý: `Fetch_Trading_Data`
- Hỗ trợ lấy dữ liệu bằng `get_data()` và dừng kết nối bằng `stop()`

**Phương thức:**

| Tên phương thức | Mô tả |
|---|---|
| get_data() | Nhận dữ liệu lasted nếu realtime = False hoặc bắt đầu kết nối và nhận dữ liệu với realtime = True |
| stop() | Dừng kết nối |

**Các thuộc tính của dữ liệu:**

| Tên thuộc tính | Mô tả | Kiểu dữ liệu |
|---|---|---|
| ticker | Tên mã | str |
| timestamp | Thời gian giao dịch | int |
| open | Giá mở cửa | float |
| low | Giá thấp nhất | float |
| high | Giá cao nhất | float |
| close | Giá đóng cửa | float |
| volume | Khối lượng giao dịch | int |
| value | Giá trị giao dịch | int |
| bu | Khối lượng mua chủ động toàn thị trường | int |
| sd | Khối lượng bán chủ động toàn thị trường | int |
| fb | Giá trị bán khối ngoại | int |
| fs | Giá trị mua khối ngoại | int |
| fn | Giá trị mua/bán ròng | int |

**Ví dụ sử dụng 1:**

*Trường hợp 1: Sử dụng from_date*

```python
import time
import pandas as pd
from FiinQuantX import FiinSession, BarDataUpdate

username = 'REPLACE_WITH_YOUR_USER_NAME'
password = 'REPLACE_WITH_YOUR_PASS_WORD'

client = FiinSession(username=username, password=password).login()

tickers = ['HPG', 'SSI', 'VNM', 'VIC', 'VRE']

def onUpdate(data: BarDataUpdate):
    print(data.to_dataFrame())
    print('-------------Callback-------------')
    
event = client.Fetch_Trading_Data(
    realtime = True,
    tickers = tickers,    
    fields = ['open','high','low','close','volume','value','bu','fn'],
    by = '1m', 
    callback = onUpdate, 
    from_date='2024-11-28 09:00',
    wait_for_full_timeFrame = False
)

event.get_data()

try:
    while not event._stop:
        time.sleep(1)
except KeyboardInterrupt:
    print("KeyboardInterrupt received, stopping...")
    event.stop()
```

**Ví dụ sử dụng 2:**

*Trường hợp 2: Sử dụng period*

```python
import time
import pandas as pd
from FiinQuantX import FiinSession, BarDataUpdate

username = 'REPLACE_WITH_YOUR_USER_NAME'
password = 'REPLACE_WITH_YOUR_PASS_WORD'

client = FiinSession(username=username, password=password).login()

tickers = ['HPG', 'SSI', 'VN30', 'VN30F1M', 'VRE']

def onUpdate(data: BarDataUpdate):
    print(data.to_dataFrame())
    print('-------------Callback-------------')
    
event = client.Fetch_Trading_Data(
    realtime = True,
    tickers = tickers,    
    fields = ['open', 'high', 'low', 'close', 'volume', 'value', 'bu'],
    by = '1m', 
    period = 10,
    callback = onUpdate, 
    wait_for_full_timeFrame=False
)

event.get_data()
time.sleep(3600)
event.stop()
