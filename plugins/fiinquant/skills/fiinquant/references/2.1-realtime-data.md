# 2.1. Hàm dữ liệu Realtime

Mô tả cách sử dụng thư viện sau khi người dùng đã đăng nhập. Các ví dụ chi tiết được nêu ra ở cuối chương này.

**Tên hàm:** `client.Trading_Data_Stream(tickers=tickers, callback=callback)`

**Mục đích:**
Nhận dữ liệu giao dịch realtime cho một hoặc nhiều mã chứng khoán, chỉ số, phái sinh hoặc chứng quyền. Dữ liệu sẽ được xử lý thông qua hàm callback do người dùng tự định nghĩa.

Hàm này tạo realtime stream chạy dài hạn. Ứng dụng phải quản lý rõ thời điểm `start()`, `stop()` và trách nhiệm reconnect. Xem [Quản lý kết nối và tối ưu hiệu năng](2.6-connection-lifecycle-performance.md) trước khi triển khai cho nhiều mã hoặc ứng dụng chạy liên tục.

**Tham số (Parameters):**

| Tên tham số | Mô tả | Kiểu dữ liệu | Mặc định | Bắt buộc |
|---|---|---|---|---|
| tickers | Tên mã, bao gồm mã chứng khoán, mã chỉ số, mã phái sinh và mã chứng quyền. Các mã này được viết in hoa. | str hoặc list |  | Có |
| callback | Là hàm do người dùng tự định nghĩa để thao tác với dữ liệu. | function | None | Có |

**Kết quả trả về (Returns):**
- Kiểu dữ liệu: object stream dữ liệu realtime
- Class nhận dữ liệu là `Trading_Data_Stream`
- Có thể sử dụng các phương thức như `start()` và `stop()` để điều khiển luồng dữ liệu
- Hỗ trợ truy xuất từng thuộc tính dữ liệu hoặc chuyển toàn bộ dữ liệu sang DataFrame bằng phương thức `to_dataFrame()`

**Hàm callback**

Là phương thức do người dùng tự định nghĩa để thao tác với dữ liệu, sẽ được truyền vào khi khởi tạo đối tượng nhận dữ liệu như là một đối số. Phương thức này sẽ có dạng như sau:

```python
# pseudocode
def name_of_callback(data: RealTimeData):
    # do something

Events = client.Trading_Data_Stream(
    tickers=['ticker_1', 'ticker_2', 'ticker_n'],
    callback=name_of_callback
)
Events.start()
```

## Quản lý vòng đời realtime stream

- Một ứng dụng nên dùng một luồng chung cho tập hợp các mã đang cần realtime, thay vì tạo luồng riêng theo từng mã hoặc từng tính năng.
- Luôn gọi `stop()` trong khối `finally` để đóng stream khi chương trình dừng hoặc gặp exception.
- Không dùng private attribute như `_stop`, `thread` hoặc `hub_connection` để kiểm tra hay điều khiển stream.
- Khi cần thay đổi tập mã nhưng SDK không cung cấp API cập nhật subscription, phải đóng hoàn toàn stream cũ trước khi tạo stream mới.
- Không chạy nhiều reconnect loop cho cùng một stream. Chỉ một thành phần được chịu trách nhiệm reconnect.

**Ví dụ:**

```python
import time

from FiinQuantX import FiinSession, RealTimeData

username = 'REPLACE_WITH_YOUR_USER_NAME'
password = 'REPLACE_WITH_YOUR_PASS_WORD'

client = FiinSession(username=username, password=password).login()

tickers = ['HPG', 'VNINDEX', 'VN30F1M']

def onTickerEvent(data: RealTimeData):
    print('----------------')
    print(data.to_dataFrame())
    # data.to_dataFrame().to_csv('callback.csv', mode='a', header=True)

events = client.Trading_Data_Stream(
    tickers=tickers,
    callback=onTickerEvent
)
events.start()

try:
    while True:
        time.sleep(1)
except KeyboardInterrupt:
    print("KeyboardInterrupt received, stopping...")
finally:
    events.stop()
```
