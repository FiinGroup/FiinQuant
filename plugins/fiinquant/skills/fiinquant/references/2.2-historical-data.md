# 2.2. Hàm dữ liệu Lịch sử

Mô tả cách sử dụng thư viện sau khi người dùng đã đăng nhập. Các ví dụ chi tiết được nêu ra ở cuối chương này.

**Tên hàm:** `client.Fetch_Trading_Data(...).get_data()`

**Mục đích:**
Lấy dữ liệu giao dịch lịch sử (OHLCV và các trường liên quan) hoặc realtime dạng nến cho một hoặc nhiều mã chứng khoán, phục vụ phân tích kỹ thuật, backtest hoặc hiển thị dữ liệu.

## Vòng đời historical request

Với `realtime=False`, `get_data()` thực hiện historical request theo cơ chế request-response và trả về dữ liệu. Chế độ này không khởi tạo realtime stream, vì vậy không gọi `stop()` sau `get_data()`.

Với `realtime=True`, đối tượng tiếp tục duy trì realtime stream sau khi tải dữ liệu ban đầu. Trường hợp này phải gọi `stop()` khi kết thúc. Xem [Hàm nối dữ liệu Realtime và lịch sử](2.3-merge-realtime-historical.md) và [Quản lý kết nối và tối ưu hiệu năng](2.6-connection-lifecycle-performance.md).

**Tham số (Parameters):**  
*(Lưu ý: `period` chỉ tồn tại khi không truyền `from_date` và ngược lại. Phải chọn 1 trong 2 cách truyền.)*

| Tên tham số | Mô tả | Kiểu dữ liệu | Mặc định | Bắt buộc |
|---|---|---|---|---|
| realtime | Có đăng ký vào dữ liệu update liên tục hay không hay chỉ gọi dữ liệu lịch sử đến thời điểm mới nhất (True là có, False là không). | bool |  | Có |
| tickers | Danh sách mã được viết in hoa. | str hoặc list |  | Có |
| fields | Các trường dữ liệu cần lấy ['open','high','low','close','volume','bu','sd','fb','fs','fn'] | list |  | Có |
| adjusted | Giá điều chỉnh hay chưa điều chỉnh (True là đã điều chỉnh và ngược lại). | bool | True | Không |
| by | Đơn vị thời gian (1m, 5m, 15m, 30m, 1h, 2h, 4h, 1d) | str | 1M | Không |
| period | Số nến lịch sử gần nhất cần lấy | int |  | Không |
| from_date | Mốc thời gian lấy dữ liệu xa nhất | str | datetime | Không |
| to_date | Mốc thời gian lấy dữ liệu gần nhất | str | datetime.now() | Không |
| lasted | Có lấy dữ liệu nến mới nhất chưa đóng hay không | bool | False | Không |

**Best Practices:**
Nếu muốn lấy dữ liệu mới nhất cho ngày hôm nay để thực hiện phân tích thị trường trong ngày mới nhất, buộc phải để biến `lasted=True`. Ví dụ muốn phân tích các cổ phiếu HPG, SSI, VN30F1M, UPCOMINDEX trong ngày hôm nay:
```python
import pandas as pd
from FiinQuantX import FiinSession

username = 'REPLACE_WITH_YOUR_USER_NAME'
password = 'REPLACE_WITH_YOUR_PASS_WORD'

client = FiinSession(username=username, password=password).login()

tickers = ['HPG', 'SSI', 'VN30F1M', 'UPCOMINDEX']
    
data = client.Fetch_Trading_Data(
    realtime = False,
    tickers = tickers,    
    fields = ['open', 'high', 'low', 'close', 'volume', 'value', 'bu', 'sd', 'fb', 'fs', 'fn'],
    adjusted=True,
    by = '1d', 
    period=100,
    lasted=True
).get_data()

print(data)
```

### Tối ưu khi gọi dữ liệu thường xuyên

- Tái sử dụng một `FiinSession` trong vòng đời process hoặc worker; không đăng nhập lại theo từng mã.
- Giới hạn số historical request chạy đồng thời bằng queue hoặc semaphore phù hợp với tải hệ thống.
- Gộp nhiều mã trong một request khi các mã dùng cùng timeframe, fields và khoảng thời gian.
- Cache kết quả theo ticker, timeframe, fields, adjusted và phạm vi dữ liệu.
- Dedupe các request đang chạy có cùng tham số.
- Với giao diện tương tác, chỉ tải lượng dữ liệu cần cho lần hiển thị đầu tiên và tải phần cũ hơn theo trang khi cần.
- Khi người dùng đổi mã hoặc timeframe, không để kết quả của request cũ ghi đè trạng thái hiện tại.


**Kết quả trả về (Returns):**
- Kiểu dữ liệu: dữ liệu bảng (thường là DataFrame)
- Class xử lý: `Fetch_Trading_Data`
- Phương thức chính: `get_data()` để lấy dữ liệu

**Phương thức của class Fetch_Trading_Data**

| Tên phương thức | Mô tả |
|---|---|
| get_data() | Trả dữ liệu historical nếu `realtime=False`; tải dữ liệu ban đầu và bắt đầu realtime stream nếu `realtime=True` |

**Các thuộc tính của dữ liệu:**

| Tên thuộc tính | Mô tả | Kiểu dữ liệu |
|---|---|---|
| ticker | Tên mã | str |
| timestamp | Thời gian giao dịch (Thời gian giao dịch, kiểu chuỗi datetime string theo định dạng "%Y-%m-%d %H:%M"). Nếu by='1d' thì sẽ có dạng "%Y-%m-%d 00:00" còn nếu by timeframe bé hơn '1d' sẽ có dạng "%Y-%m-%d %H:%M" chi tiết theo phút | str |
| open | Giá mở cửa | float |
| low | Giá thấp nhất | float |
| high | Giá cao nhất | float |
| close | Giá đóng cửa | float |
| volume | Khối lượng giao dịch | int |
| value | Giá trị giao dịch | int |
| bu | Khối lượng mua chủ động toàn thị trường | int |
| sd | Khối lượng bán chủ động toàn thị trường | int |
| fb | Giá trị mua khối ngoại | int |
| fs | Giá trị bán khối ngoại | int |
| fn | Giá trị mua/bán ròng | int |

**Ví dụ sử dụng:**

```python
import pandas as pd
from FiinQuantX import FiinSession

username = 'REPLACE_WITH_YOUR_USER_NAME'
password = 'REPLACE_WITH_YOUR_PASS_WORD'

client = FiinSession(username=username, password=password).login()

tickers = ['HPG', 'SSI', 'VN30F1M', 'UPCOMINDEX']
    
data = client.Fetch_Trading_Data(
    realtime = False,
    tickers = tickers,    
    fields = ['open', 'high', 'low', 'close', 'volume', 'value', 'bu', 'sd', 'fb', 'fs', 'fn'],
    adjusted=True,
    by = '1d', 
    from_date='2024-11-28',
).get_data()

print(data)
```
